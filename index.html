Visão Geral do Projeto

Este documento interativo detalha a entrega do projeto Orderly, um sistema de e-commerce construído sobre uma arquitetura de microsserviços. O objetivo é demonstrar a implementação de um sistema robusto, modular e escalável. Navegue pelas seções para entender o fluxo de um pedido ou as instruções de execução.

Estrutura do Monorepo

O projeto foi desenvolvido utilizando um **monorepo**, que centraliza todos os microsserviços e bibliotecas compartilhadas, facilitando o gerenciamento de dependências e a execução do sistema completo.

Fluxograma de Processamento de um Pedido

O fluxo de um pedido é assíncrono e resiliente, orquestrado pelo `order-service` e seguindo uma sequência clara de validações para garantir a consistência dos dados.

1. Cliente

Inicia o pedido via API REST

2. Order Receiver

Recebe, valida e publica na fila

3. RabbitMQ

Pedido aguarda em uma fila

4. Order Service

Consome da fila e inicia a orquestração

Orquestração de Validações

4a. Validação de Estoque

Comunicação com Stock Service

4b. Processamento de Pagamento

Comunicação com Payment Service

5. Atualização de Status Final

Instruções de Execução

Para executar o projeto, siga os passos abaixo. É necessário ter **JDK 21**, **Maven** e **Docker** instalados.

Passo 1: Clonar o Repositório

Use o comando abaixo para clonar o repositório do projeto para sua máquina local.

bash 
git clone [URL_DO_SEU_REPOSITORIO]
cd orderly

Passo 2: Construir todos os Módulos

Execute o comando a seguir na raiz do monorepo para compilar o projeto e instalar as dependências.

bash 
mvn clean install

Passo 3: Subir a Infraestrutura (Recomendado)

Utilize o Makefile para iniciar todos os contêineres de forma integrada.

bash 
# Sobe todos os serviços em modo detached
make up

# Para derrubar todos os serviços
make down

Testes e Qualidade de Código

A qualidade do código é garantida por testes unitários e uma cobertura mínima de 80%, aferida com JaCoCo.

Executar Testes e Gerar Relatórios

Para rodar os testes de todos os módulos e gerar os relatórios de cobertura, utilize o comando do Makefile na raiz do projeto.

bash 
make test-coverage

Visualizar Relatório Centralizado

Após a execução dos testes, um relatório HTML centralizado é gerado. Abra o seguinte arquivo no seu navegador para visualizar a cobertura de cada microsserviço:

target/jacoco-reports-centralized/index.html
